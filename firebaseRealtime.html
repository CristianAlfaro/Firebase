Firebase Realtime Database

Firebase Realtime Database es una base de datos NoSQL alojada en la nube. se sincronizan en tiempo real con cada cliente conectado. todos los clientes comparten una instancia de Realtime Database y reciben actualizaciones automáticamente con los datos más recientes.

Funciones

  • Tiempo Real: cada vez que cambian los datos, los dispositivos conectados reciben esa actualización en milisegundos

  • Sin Conexiones: Firebase continúa respondiendo, aún sin conexión, guardando los datos en la memoria interna, cuando se restablece la conexión, el dispositivo recibe los cambios y los sincroniza con el estado actual del servidor.

  • Acceso desde dispositivos cliente: Firebase Realtime Database puede acceder directamente a la información sin la necesidad de la conexión a una API, ya que Android Studio lo implementa de forma interna con sus propios métodos.

  • Escalamiento en varias bases de datos: F.R.D. puede dividir la información en diferentes instancia dentro de bases de datos en el mismo proyecto, un ejemplo de estos sería poder autenticar a usuarios en diferentes instancias.


F.R.D. proporciona un lenguaje flexible de reglas basadas en expresiones, llamado reglas de seguridad de Firebase Realtime Database, para definir cómo se deberían estructurar los datos y en qué momento se pueden leer o escribir.

La API de R.D. está diseñada para permitir solo operaciones que se puedan ejecutar rápidamente. Eso permite crear una excelente experiencia de tiempo real que puede servir a millones de usuarios sin afectar la capacidad de respuesta.
